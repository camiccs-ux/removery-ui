# Removery UI

Welcome to Removery UI — a React component library and design token catalog that mirrors the **Removery Library** in Figma, built during the design system audit of the R7 Scheduling module. It's structured like shadcn/ui: components are hand-written, own-your-code building blocks (Radix primitives + Tailwind), not a black-box package.

## What's Inside

### Foundations

Design decisions that underpin every component:

- **Colors** — brand palette from the `🌗 Theme Mode` collection, including the R7 scheduling status tokens (`status-new`, `status-pending`, `status-confirmed`, `status-checked-in`, `status-in-service`, `status-noshow`, `status-blocked`)
- **Typography** — font families (Archivo + ABC Social Extended), confirmed sizes/weights/line-heights read directly from Figma
- **Dark Mode** — CSS variable system and how to activate it
- **Accessibility** — WCAG 2.1 AA contrast targets, keyboard nav, ARIA patterns, and a testing checklist
- **Spacing** — gap/padding scale and border radius, confirmed from the Figma `Tailwind Primitives` and `🌗 Theme Mode` collections

Motion and Grid & Layout foundations aren't documented yet — they're on the roadmap, not skipped by mistake.

### Components

33 components confirmed against the Removery Library audit, organized by function:

| Category | Components |
|---|---|
| **Data Display** | Accordion, Avatar, DataTable, Progress, Skeleton, Table |
| **Feedback** | Alert |
| **Layout** | Hover Card, Sidebar |
| **Navigation** | Breadcrumb, Context Menu, Dropdown Menu, Menubar, Navigation Menu, Pagination, Tabs |
| **Components** *(flat — no subfolder, everything else lives directly here)* | Avatar Group, Badge, Button, Calendar, Card, Checkbox, Command, Dialog, Drawer, Input, Label, Popover, Radio Group, Select, Separator, Sheet, Switch, Textarea, Tooltip |

**Data Display**, **Feedback**, **Layout**, **Navigation**, and **Components** are five separate top-level collapsible groups in the sidebar (siblings, not nested inside one another) — everything that doesn't fit the first four sits flat inside **Components**, by design. `Menubar` and `Command` don't map cleanly to any of the 4 collapsible groups either — `Menubar` stayed in Navigation since it's a menu-family component, `Command` moved into the flat Components list.

`DataTable` and `Table` are intentionally both listed: `Table` demonstrates the raw primitives (`TableHeader`, `TableRow`, `TableCell`, etc. — all from the same `table.tsx` file), while `DataTable` shows the composed R7 pattern (status badges, studio/time columns) built on top of those primitives. There's no separate `data-table.tsx` file — both stories import from `table.tsx`.

This isn't a full 1:1 mirror of the Removery Library yet — it's every component confirmed and mapped so far in the audit (see the *Removery Library vs. R7 Usage* audit doc).

### Not found in Figma (absence isn't proof — just unconfirmed)

`Spinner`, `Banner`, `Stack`, `Grid`, `ScrollArea`, `AspectRatio`, `ResizablePanels`, `FileUpload`, `Stepper`, `TreeView`, `Timeline`, `Meter`, `SegmentedControl`, and `DateRangePicker`/`TimePicker` didn't turn up in single-term searches of the library — per the project's own search convention, that doesn't confirm they don't exist, only that they weren't found this way. `Spinner` is worth building regardless of Figma status; it's a standard loading pattern most apps need and costs little to add.

### Naming consolidation — don't build duplicates

A few items from common design-system templates would just duplicate something Removery UI already has, or duplicate each other:

- **IconButton** — already covered by `Button` with `variant="ghost" size="icon"` (see `SidebarTrigger` for a real usage example). No separate component needed.
- **Divider** — already covered by `Separator` (same shadcn/Radix pattern, just a different name). Keep `Separator`, don't add `Divider`.
- **ToggleSwitch** — already covered by `Switch`. Same component, different name.
- **PasswordInput / SearchInput / NumberInput** — these are visual/behavioral variants of `Input` (a show/hide icon, a search icon + clear button, spinner controls), not structurally different components. Worth composing on top of `Input` rather than building three new base components.
- **PinInput / TagInput / DateRangePicker / TimePicker** — these *do* have genuinely different interaction models (segmented input, multi-value chips, dual calendars, time wheels) and are worth building as their own components later — just not as thin `Input` wrappers.
- **Chip / NotificationBadge** vs **Badge / Tag** — heavy overlap. Recommend confirming a real distinct use case in R7 before adding either; otherwise `Badge` (status/flags) and `Tag` (removable, once built) should cover it.

## Getting Started

### How this library works

There's no published npm package (yet) — like shadcn/ui, you own the code. Copy the component file(s) you need from `src/components/ui/` into your own project, along with their Radix peer dependency.

### Local setup

```
# 1. Install dependencies
npm install

# 2. Run Storybook
npm run storybook

# 3. Run the app shell (optional, for manual QA)
npm run dev
```

### Tailwind config

This project uses **Tailwind v4** — tokens live directly in `src/index.css` under `@theme inline`, no `tailwind.config.js` needed. Path alias `@/*` is set up in `vite.config.ts`, `tsconfig.json`, and `components.json`, matching shadcn/ui conventions.

### Import styles

```tsx
// main.tsx
import './index.css'
```

### Load fonts

Archivo (`font/body-text`) is available on Google Fonts:

```html
<link rel="preconnect" href="https://fonts.googleapis.com" />
<link rel="preconnect" href="https://fonts.gstatic.com" crossorigin />
<link href="https://fonts.googleapis.com/css2?family=Archivo:wght@400;500;600;700&display=swap" rel="stylesheet" />
```

ABC Social Extended (`font/display-text`) is a **licensed font** (ABC Dinamo) — it isn't on Google Fonts. The font files need to be added to the project and self-hosted via `@font-face` before it renders correctly; until then it falls back to Archivo.

### Use a component

```tsx
import { Button } from '@/components/ui/button'

<Button variant="pill">Book Free Consultation</Button>
```

For color and typography values, see **Foundations → Colors** and **Foundations → Typography**. Color hex values are currently visual approximations pending sync with the published Figma variables — see the note at the top of `src/index.css`.
