# Removery UI

Welcome to Removery UI — a React component library and design token catalog that mirrors the **Removery Library** in Figma, built during the design system audit of the R7 Scheduling module. It's structured like shadcn/ui: components are hand-written, own-your-code building blocks (Radix primitives + Tailwind), not a black-box package.

## What's Inside

### Foundations

Design decisions that underpin every component:

- **Colors** — brand palette from the `🌗 Theme Mode` collection, including the R7 scheduling status tokens (`status-new`, `status-pending`, `status-confirmed`, `status-checked-in`, `status-in-service`, `status-noshow`, `status-blocked`)
- **Typography** — font families (Archivo + ABC Social Extended), confirmed sizes/weights/line-heights read directly from Figma
- **Dark Mode** — CSS variable system and how to activate it
- **Accessibility** — WCAG 2.1 AA contrast targets, keyboard nav, ARIA patterns, and a testing checklist

Spacing, Motion, and Grid & Layout foundations aren't documented yet — they're on the roadmap, not skipped by mistake.

### Components

31 components confirmed against the Removery Library audit, organized by function:

| Category | Components |
|---|---|
| **Actions** | Button |
| **Data Entry** | Input, Label, Textarea, Checkbox, Radio Group, Switch, Select, Calendar |
| **Data Display** | Card, Badge, Avatar, Avatar Group, Progress, Skeleton, Accordion, Data Table |
| **Feedback** | Alert, Dialog, Sheet, Drawer, Tooltip, Popover, Hover Card |
| **Navigation** | Tabs, Breadcrumb, Pagination, Dropdown Menu, Context Menu, Menubar, Navigation Menu, Command |

This isn't a full 1:1 mirror of the Removery Library yet — it's every component confirmed and mapped so far in the audit (see the *Removery Library vs. R7 Usage* audit doc). Anything still missing from the Figma library can be added following the same pattern used here.

## Getting Started

### How this library works

There's no published npm package (yet) — like shadcn/ui, you own the code. Copy the component file(s) you need from `src/components/ui/` into your own project, along with their Radix peer dependency.

### Local setup

```
# 1. Install dependencies
npm install

# 2. Run Storybook
npm run storybook

# 3. Run the app shell (optional, for manual QA)
npm run dev
```

### Tailwind config

This project uses **Tailwind v4** — tokens live directly in `src/index.css` under `@theme inline`, no `tailwind.config.js` needed. Path alias `@/*` is set up in `vite.config.ts`, `tsconfig.json`, and `components.json`, matching shadcn/ui conventions.

### Import styles

```tsx
// main.tsx
import './index.css'
```

### Load fonts

Archivo (`font/body-text`) is available on Google Fonts:

```html
<link rel="preconnect" href="https://fonts.googleapis.com" />
<link rel="preconnect" href="https://fonts.gstatic.com" crossorigin />
<link href="https://fonts.googleapis.com/css2?family=Archivo:wght@400;500;600;700&display=swap" rel="stylesheet" />
```

ABC Social Extended (`font/display-text`) is a **licensed font** (ABC Dinamo) — it isn't on Google Fonts. The font files need to be added to the project and self-hosted via `@font-face` before it renders correctly; until then it falls back to Archivo.

### Use a component

```tsx
import { Button } from '@/components/ui/button'

<Button variant="pill">Book Free Consultation</Button>
```

For color and typography values, see **Foundations → Colors** and **Foundations → Typography**. Color hex values are currently visual approximations pending sync with the published Figma variables — see the note at the top of `src/index.css`.
