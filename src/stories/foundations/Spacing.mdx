# Spacing

The spacing scale keeps padding, gaps, and border radius consistent across components. All values on this page are ✅ **Figma-confirmed**, sourced from `tokens.json` — the real export of the **"Tailwind Primitives"** collection (padding/gap/border-radius) and the **"🌗 Theme Mode"** collection (semantic radius) in the Removery Library.

<div style={noteStyle}>
  <strong>Note:</strong> The base 4px-unit scale matches Tailwind's own defaults, but the collection extends well past Tailwind's stock scale — steps like <code style={codeStyle}>gap-13</code> (52px), <code style={codeStyle}>gap-17</code> (68px), <code style={codeStyle}>p-30</code> (120px), and <code style={codeStyle}>p-48</code> (192px) don't exist in Tailwind out of the box. This is a genuinely extended Removery-specific scale, not a 1:1 mirror as previously documented.
</div>

## Border Radius — semantic scale (🌗 Theme Mode)

This is the layer that's Removery-specific in name (it lives in the theme-aware collection rather than the raw primitives one) — but confirmed values show it's currently a **direct 1:1 alias** of the raw primitives above, not an independently-tuned scale. Every value matches exactly.

`src/index.css` implements this with the standard shadcn single-base-variable pattern: one `--radius` value drives `sm`/`md`/`lg`/`xl` via `calc()`, so a future rebrand only requires changing one number. The base is now set to `0.5rem` (8px = `rounded-lg`), which makes the calc-derived steps land exactly on the confirmed values:

## Z-index

No z-index variables exist in the Removery Library — layering isn't tokenized in Figma today. In code, overlay components (Dialog, Sheet, Drawer, Popover, Dropdown/Context Menu) use hardcoded Tailwind utilities rather than a shared scale:

If layering conflicts start showing up as more overlay components are combined, this is the point where a proper `--z-*` token scale should get added — it doesn't exist in Figma to copy from, so the values would need to be defined here first.
