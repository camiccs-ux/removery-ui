# Spacing

The spacing scale keeps padding, gaps, and border radius consistent across components. Unlike the color and status tokens (which are custom Removery values), the tables below come from a Figma variable collection literally named **"Tailwind Primitives"** — it mirrors Tailwind's own default scale 1:1, so what you see in Figma should match the utility classes in code exactly.

<div style={noteStyle}>
  <strong>Note:</strong> These are raw Tailwind primitives, not custom Removery tokens — <code style={codeStyle}>gap-4</code> is Tailwind's own 16px step, not a Removery-specific value like <code style={codeStyle}>status-confirmed</code>. Border radius has a second, semantic layer on top (see below) that <em>is</em> Removery-specific.

## Border Radius — semantic scale (🌗 Theme Mode)

This is the layer that's actually Removery-specific: a `radius/rounded-*` scale in the `🌗 Theme Mode` collection, computed from a single base radius rather than raw Tailwind steps — the same pattern this project already uses in `src/index.css` (`--radius` as the base, `--radius-sm/md/lg/xl` derived via `calc()`).

Confirmed **names** in Figma: `rounded-none`, `rounded-xs`, `rounded-sm`, `rounded-md`, `rounded-lg`, `rounded-xl`, `rounded-2xl`, `rounded-3xl`, `rounded-4xl`, `rounded-6xl`, `rounded-full`. Their **pixel values** aren't confirmed yet (same limitation as the Typography scale — no batch lookup, needs a text/shape layer per token to inspect).

This project's `src/index.css` today only defines 4 of those 11 steps:

<SimpleTable
  columns={['Token in index.css', 'Value']}
  rows={[
    ['--radius (base)', '0.625rem (10px)'],
    ['--radius-sm', 'calc(var(--radius) - 4px) = 6px'],
    ['--radius-md', 'calc(var(--radius) - 2px) = 8px'],
    ['--radius-lg', 'var(--radius) = 10px'],
    ['--radius-xl', 'calc(var(--radius) + 4px) = 14px'],
  ]}
/>

`rounded-none`, `rounded-2xl`, `rounded-3xl`, `rounded-4xl`, and `rounded-6xl` exist in Figma but have no equivalent in this codebase yet — that's a real gap, not an omission in this doc.

## Z-index

No z-index variables exist in the Removery Library — layering isn't tokenized in Figma today. In code, overlay components (Dialog, Sheet, Drawer, Popover, Dropdown/Context Menu) use hardcoded Tailwind utilities rather than a shared scale:

<SimpleTable
  columns={['Value in use', 'Where']}
  rows={[
    ['z-50', 'Dialog, Sheet, Drawer overlays and content, Dropdown/Context Menu, Popover, Tooltip, Command'],
    ['z-20', 'Sidebar rail'],
    ['z-10', 'Sidebar container'],
    ['z-0 / z-[1]', 'Navigation Menu indicator, avatar stacking in Avatar Group'],
  ]}
/>

If layering conflicts start showing up as more overlay components are combined, this is the point where a proper `--z-*` token scale should get added — it doesn't exist in Figma to copy from, so the values would need to be defined here first.
