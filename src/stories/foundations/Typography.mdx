# Typography

Removery UI uses two font families for visual hierarchy. All values on this page are ✅ **Figma-confirmed** — the composite styles were read directly from named text layers (`get_design_context`), and the full size/weight/line-height/tracking scale below comes from `tokens.json`, the real export of the **"Tailwind Typography"** collection in the Removery Library.

## Font Families

<SimpleTable
  mono

  rows={[

`font-mono-text`, `font-serif-text`, and `font-caveat-text` also exist as variables in the Removery Library — confirmed in `tokens.json` — but all three currently resolve to **"Archivo"** as well, i.e. they aren't yet differentiated from body text at the Figma variable level. Not included as distinct rows above since there's no visual difference to document yet.

## Scale pending confirmation

The library has 6 more `display` steps (`sm`, `md`, `lg`, `xl`, `2xl`, `3xl`) and the full `body` scale (`text-xs`, `text-sm`, `text-lg`, `text-xl`, each in `normal`/`semibold`/`bold`) as named **composite text-layer styles** — same idea as `body/text-base/semibold` above. These are different from the raw size/weight/line-height primitives above (which are now fully confirmed): a composite style is a specific combination applied to a specific named layer, and each one still needs its own layer link to confirm which primitives it actually uses.

If you want to close out the full composite-style scale, send the link to another text layer (Copy link to selection, on a layer styled `display/lg` or `body/text-xs/normal`, for example) and this page will keep growing.
