# Typography

Values confirmed directly from the Removery Library Figma file via `get_design_context` — not approximated. Unlike the colors (still pending sync), the tokens below were already read from the real Figma source.

## Font families

The Removery Library defines 5 family variables (`font/body-text`, `font/display-text`, `font/mono-text`, `font/serif-text`, `font/caveat-text`) in the `🌗 Theme Mode` collection. Today, only the first two are actually used — `mono-text`, `serif-text`, and `caveat-text` exist in the library but aren't applied anywhere in R7 or the public funnel.

## Scale pending confirmation

The library has 6 more `display` steps (`sm`, `md`, `lg`, `xl`, `2xl`, `3xl`) and the full `body` scale (`text-xs`, `text-sm`, `text-lg`, `text-xl`, each in `normal`/`semibold`/`bold`) — we just haven't pulled the exact px/line-height for each yet, since that requires opening one text layer per style in Figma (no batch lookup available). The token names (`font-size/text-*`, `line-height/leading-*`, `font-weight/font-*`) follow Tailwind's standard scale exactly, so it's likely the values match Tailwind's defaults — but that isn't confirmed and shouldn't be assumed until we check layer by layer.

If you want to close out the full scale, send the link to another text layer (Copy link to selection, on a layer styled `display/lg` or `body/text-xs/normal`, for example) and this table will keep growing.
