# Colors

Tokens from the **"🌗 Theme Mode"** collection in the Removery Library. Both names and hex values below are ✅ **Figma-confirmed** — sourced directly from `tokens.json`, the real export of that collection via the `variables2json` plugin (no more visual approximations). Light and dark values are both included below; toggle the Storybook theme switcher to preview.

<div style={noteStyle}>
  <strong>Two things worth a design-team look</strong> (documented, not silently changed):
  <br />1. <code style={codeStyle}>destructive</code> is a soft pale-red-on-dark-red pairing in light mode (badge/alert style) but a solid strong-red-on-white pairing in dark mode (solid-button style) — same token, different visual weight per theme.
  <br />2. <code style={codeStyle}>primary-foreground</code> on <code style={codeStyle}>primary</code> is 3.31:1 in both themes — clears the 3:1 threshold for large text/UI components, but falls short of the 4.5:1 AA requirement for normal-size text.
</div>

## About hover

There's no separate `--hover` variable — the Tailwind/shadcn convention reuses <code style={codeStyle}>accent</code> for hover states (`hover:bg-accent`). In light mode `accent` is identical to `primary` (`#009da3`), so hover states read as the brand color at full strength; in dark mode `accent` is a distinct dark teal (`#002729`).
