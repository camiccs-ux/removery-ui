# Colors

Tokens from the **"🌗 Theme Mode"** collection in the Removery Library. Names and hex values below are ✅ **Figma-confirmed** — sourced directly from `tokens.json`, the real export of that collection via the `variables2json` plugin (no more visual approximations) — except dark-mode `accent`/`accent-foreground`, flagged below, which were corrected by direct design direction rather than the (broken) export value. Light and dark values are both included below; toggle the Storybook theme switcher to preview. Re-synced 2026-07-16 — see notes below on what changed.

## About hover

There's no separate `--hover` variable — the Tailwind/shadcn convention reuses <code style={codeStyle}>accent</code> for hover states (`hover:bg-accent`). `accent` is a pale teal tint in light mode (`#e1f0f1`) — it no longer matches `primary` (`#009da3`). In dark mode `accent` is now `#1f2937` (a slate/neutral surface, per design direction) with white foreground — see the note above on why this diverges from the current `tokens.json` export.

## Not synced

Two things from `tokens.json` intentionally **not** brought into this file:

- **`🖼️ Theme` collection** (default/amber brand variants) — per earlier direction, this collection isn't published/active and shouldn't be used.
- **"opacity colors"** group inside `🌗 Theme Mode` — as of this sync, entries like `primary-10`/`primary-30`/`primary-80` now correlate with the real `primary` (`#009da3`), but the export still has no actual alpha channel (every step is the same flat hex, e.g. `background-10`/`background-20`/`background-30`/`background-40` are all `#ffffff` in light mode) — so there's no real opacity scale to sync yet, just same-color duplicates under different names. Flagged here rather than synced silently; let me know if design intends these to carry actual alpha values.
