# Dark Mode

The system uses per-class CSS variables (`.dark`), the same pattern as shadcn/ui — every token has a `:root` (light) value and a `.dark` (dark) value, defined in `src/index.css`.

## How to activate

```tsx
// On the app's root element (or a specific section):
<html className={isDarkMode ? "dark" : ""}>
```

Every component already uses semantic tokens (`bg-background`, `text-foreground`, `bg-primary`, etc.) instead of hardcoded colors — so toggling `.dark` on the root propagates through the whole app automatically, with no changes needed to individual components.

## Testing in Storybook

Use the theme switcher (sun/moon icon) in the toolbar at the top of Storybook — it applies the `.dark` class to the preview live.

## Note on the `🖼️ Theme` collection

The Removery Library has a second variable collection (`🖼️ Theme`), reserved for a possible future brand/theme — **it is not dark mode** and shouldn't be confused with it. The collection this project reflects is `🌗 Theme Mode`, which is the one published and in use today.
